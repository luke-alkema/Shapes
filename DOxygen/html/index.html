Program Introduction

The TestDog application is meant to show how constructors and destructors are used and/or called within an OO application. As well, this program is one of the first OO applications that OOP students have seen (in terms of the source code) so it also serves as a example of programming style and DOxygen commenting style)

Major DOxygen features to note are:

  • the Project / Application description and documentation found in the Main Page
  • how classes are commented using DOxygen
  • how methods are commented using DOxygen
  • the fact that you can still comment your source code (in C or C++ style) and have DOxygen ignore those comments while constructing your Project documentation

That list of major DOxygen features (this time in a numbered list) are:
